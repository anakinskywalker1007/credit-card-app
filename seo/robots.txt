# ============================================================
# NOTES FOR DEVELOPERS
# ============================================================
# 1. NEVER block /images/, /*.css$, or /*.js$ from Googlebot.
#    Blocking these prevents Google from rendering your pages correctly,
#    which tanks Core Web Vitals scores in Search Console.
#
# 2. The /*.xml$ block above is broad — if you add new XML files that
#    SHOULD be crawled (like a news sitemap), add explicit Allow rules above it.
#
# 3. For the affiliate redirect paths (e.g., /go/chase-sapphire-preferred/),
#    these should return 302 (temporary) redirects and should be blocked from
#    indexing via meta robots noindex, NOT robots.txt. robots.txt blocking
#    doesn't prevent indexing of linked pages — only crawling.
#
# 4. Crawl-delay is not respected by Googlebot. Google manages its own crawl
#    rate. Use Google Search Console > Crawl Stats to monitor and adjust via
#    the "Crawl rate" setting if needed.
#
# 5. To disallow a specific page from being indexed without blocking it in
#    robots.txt (preferred for individual pages), use:
#    <meta name="robots" content="noindex, nofollow">
#    on that page's <head> section.
